# Settings, Profile and Preferences

Settings are available through the authenticated application shell. The settings area includes account data, password changes, two-factor authentication, application updates and app information.

## Account profile

Users can edit supported profile fields for their role.

Typical editable fields include:

- first name
- last name
- email where permitted
- group for students
- university or faculty association where permitted by role and backend rules

## Password change

Users can change their password by entering the current password and a new password.

## Two-factor authentication

The settings area includes two-factor authentication controls. Users can enable or disable 2FA and, where supported, configure TOTP.

Security recommendation:

- enable TOTP for privileged roles
- store recovery codes securely
- rotate credentials after suspected compromise

## Active sessions

The shared session section displays session/security context and includes logout controls for the current device.

## Account deactivation

The account section includes deactivation flows. Deactivation requires password confirmation.

## Updates

The update section checks release information and can display release details, update progress and update actions depending on the frontend deployment mode.

## Application information

The info section shows application metadata such as app name, version, environment, platform information, author details, changelog and status system links.

## Language and theme

The frontend includes language and theme contexts. Users can switch supported languages and UI appearance through the application shell controls.
