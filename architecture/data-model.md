# Data Model

Tapiz uses a relational data model designed around institutions, users, subjects, sessions, attendance, score sheets, materials, forms, quizzes and notifications.

## Core concepts

| Concept | Purpose |
|---|---|
| Institution | Academic organization using the platform. |
| Faculty and department | Organizational structure for academic programs. |
| User | Account identity with a role-based workspace. |
| Subject | Course or academic unit managed by assistants and faculty staff. |
| Session | Scheduled teaching or attendance event. |
| Attendance | Student participation record. |
| Score sheet | Grading structure and student results. |
| Material | Teaching resource made available to users. |
| Form and quiz | Structured data collection and knowledge-check workflows. |
| Notification | User-facing message or activity signal. |

## Documentation boundary

This page intentionally documents product-level data concepts only. It does not publish database schema files, migration details, internal table names or implementation-specific source paths.
